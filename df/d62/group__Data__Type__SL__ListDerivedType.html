type data_type_sl_list::type_sl_list

Derived type containing single linked list.

Type_SL_List is a generic Single Linked List. The term generic means that the data stored in each node (link) of the list is generic: at present, the stored data can be integer (of any kinds defined in IR_Precision module), real (of any kinds defined in IR_Precision module), characters (of any length) and logical. In order to insert and retrieve data from the list nodes (and in general for list manipulation) the type bound methods must be used, because the data is not directly accessible. This is due to the internal representation of the data. In order to allow generic data, the intrinsic transfer function is used to encode all user data into the internal integer(I1P) array representation. As a consequence, if the node is directed accessed the user can obtain unpredictable results due to the encoded internal representation. The provide methods for list handling are the following:

...
type(type_sl_list):: list
...
  • insert data into list head:
    call list%puth(d=#user_data)
  • insert data into list tail:
    call list%putt(d=#user_data)
  • retrieve data from list at some node "n":
    call list%get(n=n,d=#user_data)
  • retrieve data from list head:
    call list%geth(d=#user_data)
  • retrieve data from list tail:
    call list%gett(d=#user_data)
  • deleting list head link:
    call list%delh
  • deleting list tail link:
    call list%delt
  • freeing (destroying) list:
    call list%free
  • converting list to array:
    call list%array(a=#user_array)
    Note
    For the character data, the methods get and array need a supplementary arguments, namely Nc being the length of the character variable (array element for array method).
    Array method converts the list into an array thus the list must have homogeneous data. Even if the list can have nodes with different data type the conversion to a standard array can be completed successfully only if all data nodes are of the same kind (e.g. all real or all integer or all character etc...).
